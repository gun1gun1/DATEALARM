# PRD — 잉크 연구원의 장기안정성(보관성능) 알람앱

> **버전** v1.0 · **작성 기준일** 2026-06-28 (KST) · **작성자** 수열
> **대상 실행 환경** 클로드 코드 (비개발자 솔로 앱)

---

## 1. 한 줄 목적

제품(잉크 등)의 **장기 안정성(수명·보관성능)을 정해진 주기마다 검토**하기 위해, 검토 시점이 오면 알려주고 **완료할 때까지 추적**한다.

---

## 2. 사용자 / 사용 맥락

- **사용자**: 단독 사용 (연구원 본인 1인, 로그인·계정 없음)
- **현재 방식의 문제**: 달력에 수기 표시 → 매일 아침 일일이 확인 → 번거롭고 누락 위험
- **기대 변화**: "오늘 검토할 대상"을 앱 화면에서 한눈에 보고, 매일 아침 메일로도 받아 빠뜨리지 않는다.

---

## 3. 핵심 기능 (확정 규칙)

### 3.1 프로젝트별 검토 주기 확정
- 프로젝트 등록 시 **검토 주기를 확정(고정)**한다.
- 기본 주기(복수 선택 가능): **15일 / 30일 / 60일 / 90일**
- **사용자 정의 주기(N일)** 추가 입력 가능.
- 한 프로젝트는 여러 주기를 동시에 가질 수 있다. 예) 15·30·60·90일을 모두 검토.

### 3.2 검토일 계산 — **원래 예정일(시작일) 기준**
- **시작일(등록한 검토 시작일)을 기준으로 +N일** 누적 계산한다.
- ※ "완료한 날" 기준이 **아님**. 지연되어 늦게 완료해도 다음 예정일은 시작일 기준으로 고정된다.

```
예) 시작일 = 2026-01-01, 주기 = 15일
  15일차 예정일 = 2026-01-16
  30일차 예정일 = 2026-01-31
  60일차 예정일 = 2026-03-02
  90일차 예정일 = 2026-04-01
(각 주기는 "시작일 + 해당 일수"로 독립 계산)
```

### 3.3 근무일 보정 (한국 공휴일 기준)
- 계산된 예정일이 **주말(토·일) 또는 한국 공휴일**이면 근무일로 보정한다.
- **기본 방향: 다음 근무일로 이동.**
- **예외: 연속 비근무일(주말+연휴)이 3일 이상이면 → 이전 근무일로 당김.**
  - 이유: 긴 연휴는 다음 근무일까지 너무 멀어 점검이 과도하게 밀리므로, 연휴 직전 근무일에 검토.

```
보정 로직(의사코드)
function adjustToWorkday(date):
    if isWorkday(date): return date
    block = 연속된 비근무일 구간(date 포함)의 길이   // 주말+공휴일 연속 일수
    if block >= 3:
        return 구간 직전의 가장 가까운 근무일        // 이전 근무일
    else:
        return 구간 직후의 가장 가까운 근무일        // 다음 근무일

isWorkday(date): 토·일이 아니고, 한국 공휴일 목록에 없으면 true
```

- **한국 공휴일 데이터**: 외부 API 호출 없이 **연도별 공휴일 목록을 앱 내부에 내장**(JSON 상수). 설정 화면에서 **사용자가 임시공휴일·대체공휴일을 수동 추가/삭제** 가능.

### 3.4 알람 표시 (과하지 않게 + 완료까지 추적)
- **앱 화면**: "오늘의 검토" 영역에 **오늘 + 지연(밀린) 검토를 묶음으로** 표시. 개별 팝업 난사 금지.
- **상태 추적**: **완료 처리 전까지 매일 "지연(밀린) 검토"로 계속 표시**한다(스누즈/사라짐 없음).
- **완료 처리**: 해당 검토 카드를 완료 체크 → 이력에 기록. 다른 주기 예정일은 영향받지 않음(3.2 기준).

### 3.5 메일 알림 (Google Apps Script로 분리)
- **앱 본체는 순수 브라우저(localStorage)로 유지.** 메일 자동 발송은 브라우저 단독 불가하므로 **외부 자동화로 분리**한다.
- **방식**: Google Apps Script의 **시간 기반 트리거**로 **매일 오전 08:30 (KST)** 실행.
- **수신**: `gun1gun1@monami.com`
- **메일 내용**: 그날의 "오늘 + 지연" 검토 목록(프로젝트명 / 주기 / 보정된 예정일 / 보관조건).
- **데이터 연동 방식(둘 중 택1, 구현 시 단순한 쪽 우선)**:
  1. 앱에서 일정 JSON을 내보내 **구글 드라이브 파일에 저장** → GAS가 해당 파일을 읽어 발송.
  2. (보류) 별도 동기화 없이, GAS에 일정 규칙을 동일 로직으로 복제.
- **산출물**: 앱 코드와 함께 **`gas/Code.gs` 스크립트 + 설치 가이드(README의 "메일 설정" 절)**를 제공한다.

## 5. 상태 정의

| 상태 | 조건 | 화면 표시 |
|---|---|---|
| 예정(Upcoming) | 보정 예정일 > 오늘, 미완료 | 프로젝트 상세에 표시 |
| 오늘(Today) | 보정 예정일 == 오늘, 미완료 | "오늘의 검토"에 표시 |
| 지연(Overdue) | 보정 예정일 < 오늘, 미완료 | "오늘의 검토"에 **계속** 표시 (완료 전까지) |
| 완료(Done) | done == true | 이력에만 표시 |

## 6. 화면 구성 (대략)

1. **오늘의 검토** (홈)
   - 오늘 + 지연 검토를 묶음 리스트로 표시: `프로젝트명 · n일차 · 예정일 · 보관조건` + [완료] 버튼
2. **프로젝트 목록 / 등록**
   - 신규 등록: 프로젝트명, 시작일, 주기(15·30·60·90·사용자정의 N), 보관조건 선택(+ 추가)
3. **프로젝트 상세 / 이력**
   - 해당 프로젝트의 전체 예정 일정표(주기별 예정일·보정일) + 수행 이력
4. **설정 / 백업**
   - 메일 수신주소·시간 확인, 사용자 공휴일 등록, 보관조건 관리, **JSON Export / Import**, 메일(GAS) 연동 안내

## 8. 예외 / 엣지 케이스

- **같은 날 여러 검토**: 묶음 리스트로 모두 표시(개별 팝업 금지).
- **장기 지연 누적**: 지연 항목은 사라지지 않고 완료까지 유지. 오래된 순으로 정렬.
- **연속 휴일 구간**: 3.3 임계값(연속 3일 이상 → 이전 근무일)으로 처리. 임계값은 설정에서 변경 가능.
- **주기 변경**: 기본은 "시작 전 확정". 이미 진행 중인 프로젝트의 주기 변경 시 → 미완료·미래 예정일만 재계산하고, 이미 완료된 이력은 보존(이 정책을 구현 시 명시).
- **연도 전환**: 내장 공휴일은 연도별 데이터. 데이터가 없는 미래 연도는 토·일만 비근무일로 처리하고 설정에서 보완하도록 안내.
- **시작일이 휴일**: 시작일 자체는 보정하지 않음(기준일). 계산된 각 예정일에만 근무일 보정 적용.
